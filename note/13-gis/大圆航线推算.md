

# 🌍 球面“起点 + 距离 + 方位角 → 终点经纬度”



# 向量法

## 一、问题定义

已知：

- 起点经纬度：$(\varphi_1, \lambda_1)$
- 距离：$d$
- 地球半径：$R$
- 方位角：$\theta$（正北为0°，顺时针）

求终点经纬度：$(\varphi_2, \lambda_2)$

------

## 二、核心思想（向量版）

> **球面运动 = 三维空间中的旋转**

具体来说：

- 将起点映射为单位球上的向量 $\mathbf{P}_1$
- 构造"前进方向"
- 计算旋转轴
- 将 $\mathbf{P}_1$ 绕该轴旋转 $\delta = \frac{d}{R}$

------

## 三、推导过程

### 1️⃣ 经纬度 → 三维单位向量

$$
\mathbf{P}_1 =
\begin{bmatrix}
\cos\varphi_1 \cos\lambda_1 \\
\cos\varphi_1 \sin\lambda_1 \\
\sin\varphi_1
\end{bmatrix}
$$

------

### 2️⃣ 构造局部坐标系（北 / 东）

设：

- 北方向：$\mathbf{N}$
- 东方向：$\mathbf{E}$

（推导见附录）

------

### 3️⃣ 前进方向（由方位角决定）

$$
\mathbf{D} = \mathbf{N}\cos\theta + \mathbf{E}\sin\theta
$$

------

### 4️⃣ 旋转轴

$$
\mathbf{K} = \mathbf{P}_1 \times \mathbf{D}
$$

------

### 5️⃣ 旋转（罗德里格公式）

$$
\mathbf{P}_2 = \mathbf{P}_1 \cos\delta + (\mathbf{K} \times \mathbf{P}_1) \sin\delta
$$

其中：

$$
\delta = \frac{d}{R}
$$

说明：

- 已利用 $\mathbf{K} \cdot \mathbf{P}_1 = 0$ 简化公式
- 此公式展开其实跟球面三角法是一样的

------

### 6️⃣ 转回经纬度

$$
\varphi_2 = \arcsin(z)
$$

$$
\lambda_2 = \operatorname{atan2}(y, x)
$$

------

## 四、完整 JavaScript 实现

```js
function destinationPointVec(lat1, lon1, distance, bearing) {
  const R = 6371000;

  const φ = lat1 * Math.PI / 180;
  const λ = lon1 * Math.PI / 180;
  const θ = bearing * Math.PI / 180;

  const δ = distance / R;

  // 起点向量
  const P = [
    Math.cos(φ) * Math.cos(λ),
    Math.cos(φ) * Math.sin(λ),
    Math.sin(φ)
  ];

  // 北方向
  const N = [
    -Math.sin(φ) * Math.cos(λ),
    -Math.sin(φ) * Math.sin(λ),
    Math.cos(φ)
  ];

  // 东方向（已归一化）
  const E = [
    -Math.sin(λ),
    Math.cos(λ),
    0
  ];

  // 前进方向
  const D = [
    N[0] * Math.cos(θ) + E[0] * Math.sin(θ),
    N[1] * Math.cos(θ) + E[1] * Math.sin(θ),
    N[2] * Math.cos(θ) + E[2] * Math.sin(θ)
  ];

  function cross(a, b) {
    return [
      a[1]*b[2] - a[2]*b[1],
      a[2]*b[0] - a[0]*b[2],
      a[0]*b[1] - a[1]*b[0]
    ];
  }

  // 旋转轴
  const K = cross(P, D);

  // K × P
  const KxP = cross(K, P);

  // 旋转
  const P2 = [
    P[0]*Math.cos(δ) + KxP[0]*Math.sin(δ),
    P[1]*Math.cos(δ) + KxP[1]*Math.sin(δ),
    P[2]*Math.cos(δ) + KxP[2]*Math.sin(δ)
  ];

  // 转回经纬度
  const lat2 = Math.asin(P2[2]);
  const lon2 = Math.atan2(P2[1], P2[0]);

  return {
    lat: lat2 * 180 / Math.PI,
    lon: ((lon2 * 180 / Math.PI + 540) % 360) - 180
  };
}
```

# **球面三角法**

------

## 1. 球面三角法核心公式

设起始点 $P_1(\phi_1, \lambda_1)$，目标点 $P_2(\phi_2, \lambda_2)$，方位角 $\theta$，角距离 $\delta = r/R$。

### 1.1 纬度计算（球面余弦定理）

基于包含北极点、起始点、目标点的球面三角形：

$$\sin \phi_2 = \sin \phi_1 \cos \delta + \cos \phi_1 \sin \delta \cos \theta$$

### 1.2 经度计算

根据球面正弦定理与余弦定理的组合推导，经度差 $\Delta \lambda$ 满足：

$$\Delta \lambda = \operatorname{atan2}(\sin \theta \sin \delta \cos \phi_1, \cos \delta - \sin \phi_1 \sin \phi_2)$$

------

## 2. JavaScript 实现代码

这种实现方式更为简洁，也是许多开源 GIS 库（如 Leaflet 或早期的 Movable Type Scripts）所采用的标准逻辑。

JavaScript

```js
/**
 * 球面三角法推算坐标
 * @param {number} lat - 起始纬度 (DD)
 * @param {number} lon - 起始经度 (DD)
 * @param {number} distance - 距离 (米)
 * @param {number} bearing - 方位角 (0-360)
 * @param {number} R - 地球半径 (米)
 */
function destinationSpherical(lat, lon, distance, bearing, R = 6371000) {
    const toRad = Math.PI / 180;
    const toDeg = 180 / Math.PI;

    // 1. 转弧度
    const phi1 = lat * toRad;
    const lam1 = lon * toRad;
    const theta = bearing * toRad;
    const delta = distance / R;

    // 2. 计算目标纬度 phi2
    // 使用球面余弦定理：sin(phi2) = sin(phi1)cos(delta) + cos(phi1)sin(delta)cos(theta)
    const sinPhi2 = Math.sin(phi1) * Math.cos(delta) + 
                    Math.cos(phi1) * Math.sin(delta) * Math.cos(theta);
    const phi2 = Math.asin(sinPhi2);

    // 3. 计算目标经度 lam2
    // 使用 atan2 自动处理经度差的象限问题
    const y = Math.sin(theta) * Math.sin(delta) * Math.cos(phi1);
    const x = Math.cos(delta) - Math.sin(phi1) * sinPhi2;
    const lam2 = lam1 + Math.atan2(y, x);

    // 4. 结果转换与归一化
    return {
        lat: phi2 * toDeg,
        // 经度归一化到 [-180, 180]
        lon: ((lam2 * toDeg + 540) % 360) - 180
    };
}

// 示例
const start = { lat: 31.23, lon: 121.47 }; // 上海
const dest = destinationSpherical(start.lat, start.lon, 500000, 45); // 向东北 500km
console.log(`目标点:`, dest);
```

------

## 3. 两种方法的对比（开发者视角）

| **特性**     | **球面三角法 (上文)**            | **几何/向量旋转法 (前文)**     |
| ------------ | -------------------------------- | ------------------------------ |
| **数学工具** | 球面三角恒等式                   | 3D 向量运算 / 旋转矩阵         |
| **代码量**   | 极简，适合轻量级应用             | 略多，逻辑更具通用性           |
| **计算开销** | 较小，三角函数调用较少           | 稍大，涉及更多中间向量分量     |
| **稳定性**   | 在两极极小范围内可能存在精度抖动 | 在全球范围内极度稳定，无奇点   |
| **扩展性**   | 仅限于球面                       | 易于扩展到旋转、缩放等 3D 变换 |

### 建议

- 如果你只是在做简单的**地图点标注**或**距离外推**，使用**球面三角法**（这段代码）即可，性能最优。
- 如果你正在开发 3D 地球引擎（如 **Cesium**、**Three.js**）或者需要处理复杂的**空间拓扑关系**，建议坚持使用**向量旋转法**，因为它与 3D 渲染管线的逻辑是一致的。

# 附录：北方向 / 东方向的推导

## 方法一：参数化求导（严格数学）

球面参数化：

$$
\mathbf{P}(\varphi, \lambda) =
\begin{bmatrix}
\cos\varphi \cos\lambda \\
\cos\varphi \sin\lambda \\
\sin\varphi
\end{bmatrix}
$$

------

### 北方向（对纬度求导）

$$
\mathbf{N} = \frac{\partial \mathbf{P}}{\partial \varphi}
= 
\begin{bmatrix}
-\sin\varphi \cos\lambda \\
-\sin\varphi \sin\lambda \\
\cos\varphi
\end{bmatrix}
$$

------

### 东方向（对经度求导）

$$
\mathbf{E} = \frac{\partial \mathbf{P}}{\partial \lambda}
= 
\begin{bmatrix}
-\cos\varphi \sin\lambda \\
\cos\varphi \cos\lambda \\
0
\end{bmatrix}
$$

------

### 归一化东方向

由于：

$$
|\mathbf{E}| = \cos\varphi
$$

归一化得到：

$$
\mathbf{E}_{unit} =
\begin{bmatrix}
-\sin\lambda \\
\cos\lambda \\
0
\end{bmatrix}
$$

------

## 方法二：叉乘构造（工程推荐）

设：

$$
\mathbf{P} = \text{当前位置单位向量}, \quad \mathbf{Z} = (0,0,1)
$$

------

### 东方向

$$
\mathbf{E} = \frac{\mathbf{Z} \times \mathbf{P}}{|\mathbf{Z} \times \mathbf{P}|}
$$

------

### 北方向

$$
\mathbf{N} = \mathbf{P} \times \mathbf{E}
$$

------

## 几何意义总结

- $\mathbf{P}$：法线（"上"）
- $\mathbf{N}$：沿经线（北）
- $\mathbf{E}$：沿纬线（东）

👉 构成球面局部正交基（TBN）

------

## ⚠️ 极点问题

当 $\mathbf{P} \parallel \mathbf{Z}$ 时：

$$
\mathbf{Z} \times \mathbf{P} = 0
$$

解决：

```js
if (接近极点) {
  使用 X 轴代替 Z 轴
}
```

# 附录：atan2

在计算机图形学、地理信息系统（GIS）以及机器人学中，`atan2` 是一个至关重要的函数。与标准的 $\arctan(y/x)$ 不同，它能根据 $y$ 和 $x$ 的正负号确定点所在的**全平面象限**。

### 1. 数学分段定义

`atan2(y, x)` 的数学定义通常基于标准反正切函数 $\arctan(z)$，其值域为 $(-\pi, \pi]$：

$$\operatorname{atan2}(y, x) =  \begin{cases}  \arctan\left(\frac{y}{x}\right) & \text{if } x > 0 \\ \arctan\left(\frac{y}{x}\right) + \pi & \text{if } x < 0, y \ge 0 \\ \arctan\left(\frac{y}{x}\right) - \pi & \text{if } x < 0, y < 0 \\ +\frac{\pi}{2} & \text{if } x = 0, y > 0 \\ -\frac{\pi}{2} & \text{if } x = 0, y < 0 \\ \text{undefined} & \text{if } x = 0, y = 0  \end{cases}$$

------

### 2. 关键特性与 $\arctan$ 的区别

| **特性**     | **标准 arctan(y/x)**               | **atan2(y,x)**                            |
| ------------ | ---------------------------------- | ----------------------------------------- |
| **值域**     | $(-\pi/2, \pi/2)$                  | $(-\pi, \pi]$                             |
| **象限区分** | 无法区分第一与第三象限（比值相同） | 能够唯一确定四个象限                      |
| **除零处理** | 当 $x=0$ 时未定义，程序会崩溃      | 能够处理 $x=0$ 的情况（返回 $\pm \pi/2$） |
| **几何意义** | 直线的斜率角                       | 向量 $(x, y)$ 与 $x$ 轴正方向的夹角       |

------

### 3. 几何解释

想象一个平面直角坐标系中的点 $(x, y)$：

- **第一象限 $(+,+)$**：返回 $(0, \pi/2)$
- **第二象限 $(-,+)$**：返回 $(\pi/2, \pi)$
- **第三象限 $(-,-)$**：返回 $(-\pi, -\pi/2)$
- **第四象限 $(+,-)$**：返回 $(-\pi/2, 0)$

### 4. 编程实现注记

在大多数编程语言（C++, JavaScript, Python 等）中，函数签名通常为 `atan2(y, x)`。

**注意：** $y$ 坐标通常作为第一个参数传入，这是为了模仿斜率公式 $m = y/x$ 的定义。

> **小技巧**：在处理地理经度差 $\Delta \lambda$ 时，使用 `atan2` 可以自动处理跨越 $180^\circ$ 经线（即从 $179^\circ$ 增加到 $-179^\circ$）时的正负号跳转问题，确保计算出的夹角始终位于最短弧路径上。
