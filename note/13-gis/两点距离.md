在球面上计算两点间的最短路径（即劣弧距离），最常用的数学模型是**大圆距离（Great Circle Distance）**。根据计算精度和适用场景的不同，主要有两种核心公式。

设两点坐标分别为：
* 点 1：$(\phi_1, \lambda_1)$（纬度，经度）
* 点 2：$(\phi_2, \lambda_2)$
* 地球平均半径：$R \approx 6371$ km

---

## 1. 半正矢公式 (Haversine Formula) —— 编程首选
这是目前 GIS 开发中最常用的公式，它在处理**短距离**（如几米到几百公里）时具有极高的数值稳定性，能够有效避免浮点数舍入误差导致的精度损失。

### 公式
首先计算两点的纬度差与经度差（弧度）：
$$\Delta \phi = \phi_2 - \phi_1$$
$$\Delta \lambda = \lambda_2 - \lambda_1$$

计算半正矢值 $a$：
$$a = \sin^2\left(\frac{\Delta \phi}{2}\right) + \cos \phi_1 \cdot \cos \phi_2 \cdot \sin^2\left(\frac{\Delta \lambda}{2}\right)$$

计算角距离 $c$（弧度）：
$$c = 2 \cdot \operatorname{atan2}(\sqrt{a}, \sqrt{1-a})$$

最终距离 $d$：
$$d = R \cdot c$$



---

## 2. 球面余弦定律 (Spherical Law of Cosines) —— 数学最简
这是从球面三角几何直接推导出的公式，计算量较小，但在处理极短距离（如地表 1 米左右）时，由于 $\cos$ 值非常接近 1，在某些编程环境下可能会因为浮点数精度限制产生较大误差。

### 公式
$$d = R \cdot \arccos\left( \sin \phi_1 \sin \phi_2 + \cos \phi_1 \cos \phi_2 \cos(\Delta \lambda) \right)$$

---

## 3. 原理解析：为什么是“劣弧”？



### A. 大圆路径 (Great Circle)
穿过球心并将球体平分为两半的圆称为“大圆”。连接球面上两点的最短路径，必然在这个大圆的周长上。

### B. 劣弧与优弧
一个圆上的两点将圆周分为两段弧：
* **劣弧 (Minor Arc)**：长度较短的那一段（圆心角 $< \pi$）。
* **优弧 (Major Arc)**：长度较长的那一段（圆心角 $> \pi$）。
我们通常求的“距离”默认为劣弧。上述公式中的 `atan2` 或 `arccos` 会自动返回较小的弧度值，从而确保计算结果为劣弧。

## 4. JavaScript 代码实现 (Haversine)

```javascript
function getDistance(lat1, lon1, lat2, lon2) {
    const R = 6371; // 地球半径，单位 km
    const toRad = Math.PI / 180;

    const dLat = (lat2 - lat1) * toRad;
    const dLon = (lon2 - lon1) * toRad;
    
    const phi1 = lat1 * toRad;
    const phi2 = lat2 * toRad;

    const a = Math.sin(dLat / 2) * Math.sin(dLat / 2) +
              Math.cos(phi1) * Math.cos(phi2) *
              Math.sin(dLon / 2) * Math.sin(dLon / 2);

    const c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));

    return R * c; // 返回单位 km
}
```

### 注意事项：
1.  **单位一致性**：计算前务必将角度（Degree）转换为弧度（Radian）。
2.  **精度说明**：上述公式基于完美球体。如果需要极高精度（如用于导弹制导或测绘），应使用 **Vincenty 算法**，它考虑了地球是扁球体（WGS-84 坐标系）的因素，精度可达 0.5mm，但计算逻辑要复杂得多。
