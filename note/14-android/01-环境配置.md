2. 配置http代理

   代理服务器的作用

   - `内容缓存`：缓存经常访问的网页和数据，当再次访问时可以直接从代理服务器获取，加快访问速度。
   - `绕过限制`：某些国外源在国内访问的速度极慢，通过代理可以绕过限制，实现跨地域访问。
   - `安全性`：匿名浏览，数据加密

   配置 `HTTP` 代理【`方式一`】

   - 镜像地址

     ```arduino
     腾讯： https://mirrors.cloud.tencent.com/AndroidSDK/
     阿里： https://mirrors.aliyun.com/android.googlesource.com/
     ```

   - 打开 `Settings`，搜索 `HTTP Proxy`，然后填入镜像地址，点击 `Apply`，在点击 `OK`。

     ![](.\image\http_proxy.png)

   - 重新启动 `Android Studio` 会弹出这个配置面板，可以填下面的常用代理服务器，任意选个。

     ![](.\image\http_proxy1.png)

### 3. gradle

1. 下载二进制慢，例如`gradle-8.11.1-bin.zip`

   修改`gradle-wrapper.properties`中的镜像地址为：

   ```
   distributionUrl=https\://mirrors.cloud.tencent.com/gradle/gradle-8.11.1-bin.zip
   ```

2. IDE下载源码慢，例如`gradle-8.11.1-src.zip`，这个办法不太管用，可以先手动关掉sync，然后空闲时再开启sync让它慢慢下载，这个只影响配置文件`build.gradle.kts`和`settings.gradle.kts`

   去上述镜像地址同路径下找到对应版本的src源码，手动下载，然后放在`C:\Users\[您的用户名]\.gradle\wrapper\dists\gradle-8.11.1-bin\`下的某个随机目录中，具体是包含`gradle-8.11.1-bin.zip.ok`的那个。

### 5. 重命名项目

为了将项目从 "HelloAndroid" 完全更新为 "AAAArticle"，你需要修改几个关键位置。请按照以下步骤操作：

#### 1. 修改 `settings.gradle.kts` (或 `settings.gradle`)

这个文件定义了项目的根目录名称。

*   **路径**: `AAAArticle/settings.gradle.kts`
*   **修改内容**:
    *   找到类似 `rootProject.name = "HelloAndroid"` 的行。
    *   将其修改为 `rootProject.name = "AAAArticle"`。

这一步会让 Android Studio 在项目视图中正确显示你的项目名称。

#### 2. 修改 `build.gradle.kts` (或 `build.gradle`)

这个文件定义了应用模块的配置，其中最重要的就是 `applicationId`。

*   **路径**: `AAAArticle/app/build.gradle.kts`
*   **修改内容**:
    
    *   在 `android { ... }` 代码块中，找到 `applicationId`。
    *   它的值可能是 `"com.example.helloandroid"`。
    *   修改为你想要的新的唯一标识，例如 `"com.example.aaaarticle"`。
    *   **注意**: `applicationId` 是你的应用在 Google Play 商店和设备上的唯一标识，一旦发布就不能再更改。建议使用 `com.yourdomain.appname` 的格式。
    
    修改示例：
    
    ```kotlin
    // build.gradle.kts
    android {
        // ...
        defaultConfig {
            applicationId = "com.example.aaaarticle" // 修改这里
            // ...
        }
        // ...
    }
    ```

#### 3. 修改包名 (Package Name)

这是最关键的一步，它会涉及到源代码和目录结构的更改。Android Studio 提供了安全重构的工具来完成这个操作。

* 在 Android Studio 的项目视图中，切换到 Android 视图。

* 展开 app -> java。

* 你应该会看到 com.example.helloandroid 这个包。

* 右键点击 helloandroid 这一部分。

* 选择 Refactor -> Rename。

* 在弹出的警告对话框中，选择 Rename Package。

* 输入新的名称，比如 aaaarticle。

* 点击 Refactor 按钮。

* Android Studio 会扫描整个项目，找到所有使用该包名的地方（包括你提供的 MainActivity.kt 文件顶部的 package 声明）并自动更新它们。在窗口下方会显示一个预览，确认无误后点击 Do Refactor。
  这个操作会自动修改 MainActivity.kt 文件顶部的：

  ```kotlin
  package com.example.helloandroid
  ```

#### 4. 全局修改

以上三步之后，项目中还有许多地方存在原来项目名称，可以使用编辑器（推荐vscode）全局搜索旧的 "HelloAndroid" ，全局替换为 "AAAArticle"，注意区分大小写，先将大写替换一遍，再将小写替换一遍

#### 5. 重新同步项目

重新同步项目，消除MainActivity中的报错，相关的组件会自动更新，例如HelloAndroidTheme会自动替换成AAAArticleTheme
